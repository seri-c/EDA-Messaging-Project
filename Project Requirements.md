# Overview

> An event-driven notification service that reliably processes message-related events and publishes notifications asynchronously.

This project aims to develop a service that processes events generated by a messaging system and asynchronously notifies subscribed services. By enabling asynchronous communication, services can be loosely coupled and react to message lifecycle events independently. 

The service will process the following events:
- **New Message**: A new message has been created and sent
- **Message Read**: A recipient has read a message
- **Message Acknowledged**: A recipient has acknowledged a message
- **Label Added to Message**: A label has been added to a message
- **Label Removed from Message**: A label has been removed from a message

## Functional Requirements
- The service shall consume message-related events from the messaging system
- The service shall publish notifications to all subscribed services
- The service shall support the following event types:
  - New Message
  - Message Read
  - Message Acknowledged
  - Label Added to Message
  - Label Removed from Message
- The service shall retry processing failed events automatically
- The service shall route failed messages that exceed the maximum retry count to a Dead-Letter Queue (DLQ)



## Non-Functional Requirements
- Events shall contain a unique UUID
- Message processing shall be asynchronous
- The service shall detect and ignore duplicate events
- Messages shall be delivered with low latency
